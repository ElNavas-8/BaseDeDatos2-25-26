# ¿Cuándo usar NoSQL y cuándo no?

### Usar NoSQL cuando:

- Se trabaja con datos semiestructurados.
- Cuando la escala de datos es masiva.
- Cuando el sistema requiere alta disponibilidad.
- Evolución rápida del modelo.

### NO usar NoSQL cuando:

- En escenarios donde se requieren Transacciones críticas (banca clásica)
- Integridad estricta (fuerte) de datos obligatoria.
- Modelos altamente relacionales complejos pero eficientes.

La decisión no debe basarse en tendencias tecnológicas, sino en un análisis claro de las necesidades del sistema.
